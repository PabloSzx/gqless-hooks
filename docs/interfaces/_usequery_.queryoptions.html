Optional hookId

hookId: keyof gqlessHooksPool

Inherited from QueryOptions.hookId

  • Defined in common.ts:122

Unique hook identifier used to add the hook to the hooks pool received in onCompleted event and fetchMore.

For hooks pool usage you need to specify it's types using declare global interface gqlessHooksPool anywhere in your application

example
declare global {
   interface gqlessHooksPool {
    query1: {
      data: string[];
      variables: {
        variable1: number;
      };
    };
    query2: {
      data: string;
    };
  }
}

Optional sharedCacheId

sharedCacheId: CacheKey
  • Defined in useQuery.ts:237

Shared hook cache id

In order to be able to sync different query hooks data you can specify a shared cache id between those hooks which will update each other data.

This cache id also works as in memory persistence across different hook instances of the same hook, across navigating through different pages for example.
